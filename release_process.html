Release the Nexus-6 UI

This topic describes how to deploy the Nexus-6 UI to AWS; these procedures are to be completed by the Nexus-6 Development team only.

Before You Begin

Before you begin, ensure your local development environment is set up as described in Set Up Your Local Environment.

Prepare the Release

  1. Use gitflow to start and finish a release branch (specifying the appropriate release number as the tag).
  2. Push changes to develop and master branches.
  3. Add release notes to the GitHub tag.

Deploy the UI

Complete the following steps to deploy the Nexus-6 UI:

  1. Within your local nexus-6-ui repo, check out the master branch.

  2. Activate the nexus-6-ui-venv virtual environment:
    • On Windows, run $ source <path_to_venv>/nexus-6-ui-venv/Scripts/activate.
    • On Mac, run $ source <path_to_venv>\nexus-6-ui-venv\bin\activate.

  3. Run the build-deploy.py script:

    • For the UAT release, run the following command:

      python build-deploy.py --configuration stage

    • For the production release, run the following command:

      python build-deploy.py --configuration production

      Note: You can block the UI to prevent users from using the Nexus-6 UI while database changes are being made; to block the UI, run the following command with the --block-ui CLI option.

      python build-deploy.py --configuration production --block-ui

For more information about deploying the UI, refer to Build and Deploy the UI .
